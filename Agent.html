A2A Agents

Reviewer
Analyzes git diff, reads files, identifies issues, categorizes change type, assesses risk level
Planner
Receives Reviewer output via A2A message, validates scope, decides action (Issue / PR / None), structures plan
Writer
Receives Planner directive, drafts Issue/PR with all required fields, formats structured output
Gatekeeper
Critiques draft for missing evidence, unsupported claims, test plans. Produces reflection artifact. Enforces human approval before any GitHub API call.

MCP Tools

github_get_diff — fetch branch/commit diff
github_read_file — read specific file content
github_list_files — list changed files
github_get_issue — fetch existing issue
github_get_pr — fetch existing PR
github_create_issue — create issue (gated)
github_create_pr — create PR (gated)

Patterns

🔷 Planning — Planner validates scope before action
🔶 Tool Use — Real MCP calls for all data (no fabrication)
🔷 Reflection — Gatekeeper produces critique artifact
🔶 Multi-Agent — A2A message bus between 4 agents
